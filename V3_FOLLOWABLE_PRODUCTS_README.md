# V3 可跟品产品功能说明

## 功能概述

在 `http://127.0.0.1:8001/v3` 页面新增了"🎯 可跟品产品"标签页，专门展示值得跟品的产品，包含完整的产品链接、Google Trends 趋势数据、AI 综合评分，以及一键生成 AI 调研提示词功能。

## 主要功能

### 1. 可跟品产品列表

**访问方式**: 点击页面顶部的"🎯 可跟品产品"标签

**展示字段**:
- **产品信息**: 产品名称（可点击跳转）、完整产品链接、域名、品类
- **价格**: 当前售价
- **AI 评分**: 0-100 分综合评分（颜色编码：绿色≥80，黄色≥60，蓝色<60）
- **Google Trends 评分**: 0-100 分热度评分（过去90天平均）
- **趋势方向**: 上升↗ / 平稳→ / 下降↘（带颜色标签）
- **YoY 增长**: 同比去年增长百分比（绿色为正，红色为负）
- **广告数**: FB 活跃广告数量
- **跟品建议**: 🟢立即跟 / 🟡观察（带颜色标签）
- **操作**: AI 调研按钮（一键复制完整调研提示词）

**筛选功能**:
- 最低评分设置（默认 60 分，可调整 0-100）
- 点击"刷新"按钮重新加载数据

### 2. AI 调研提示词生成

**使用方式**: 点击产品行的"AI 调研"按钮

**功能**:
1. 自动生成完整的 AI 调研提示词（1400+ 字符）
2. 一键复制到剪贴板
3. 按钮状态反馈：生成中... → ✓ 已复制

**提示词内容包含**:

#### 产品基础信息
- 产品链接
- 产品名称
- 当前售价
- 产品类目

#### 市场数据
- Google Trends 评分（0-100）
- 趋势方向（上升/平稳/下降）
- YoY 增长百分比
- 季节性判断

#### 竞争情况
- FB 活跃广告数
- 广告持续时长
- 投放独立站数量
- 主要素材类型
- 竞品价格带

#### 8 维度深度分析框架
1. **市场需求验证**: Google Trends 数据、搜索意图、长尾关键词机会
2. **竞争格局分析**: 市场饱和度、品牌壁垒、差异化空间
3. **广告投放健康度**: 持续时长、素材多样化、疲劳迹象
4. **利润空间评估**: 毛利率预估、物流成本、获客成本
5. **供应链可行性**: 供应商可得性、定制门槛、发货时效
6. **合规与风险**: FDA/FCC/专利风险、关税政策、退款风险
7. **营销难度**: 产品可视化、市场教育成本、UGC 启动难度
8. **时机判断**: 入场阶段、时间窗口、机会持续性

#### 输出格式要求
- 跟品决策（🟢立即跟 / 🟡跟进观察 / 🔴坚决不碰）
- 综合评分（X/100 分）
- 核心理由（3 条以内）
- 风险提示（最多 2 条）
- 跟品策略建议（差异化角度、定价策略、首月预算、预期 ROI）
- 替代方向建议

## API 端点

### 1. 获取可跟品产品列表

```
GET /api/v3/followable-products?min_score=60&limit=100
```

**参数**:
- `min_score`: 最低评分（默认 60）
- `limit`: 返回数量限制（默认 100）

**返回示例**:
```json
{
  "ok": true,
  "scan_date": "2026-05-18",
  "total_followable": 2,
  "min_score": 60,
  "products": [
    {
      "product_url": "https://example.com/products/product-handle",
      "product_title": "Product Name",
      "price": 199.95,
      "ai_score": 75,
      "trends_score": 82,
      "trend_direction": "上升↗",
      "yoy_growth": 45,
      "active_ads_count": 25,
      "follow_decision": "🟢立即跟",
      "domain": "example.com",
      "handle": "product-handle",
      "product_type": "Category",
      "vendor": "Brand Name",
      ...
    }
  ]
}
```

### 2. 生成 AI 调研提示词

```
POST /api/v3/generate-research-prompt?product_url=https://example.com/products/handle
```

**参数**:
- `product_url`: 完整产品链接（必填）

**返回示例**:
```json
{
  "ok": true,
  "product_url": "https://example.com/products/handle",
  "prompt": "# 产品跟品深度调研\n\n## 产品基础信息...",
  "prompt_length": 1419
}
```

## 数据字段说明

### 核心字段
- **product_url**: 产品完整链接（可直接访问）
- **product_title**: 产品标题
- **price**: 当前售价（美元）
- **domain**: 域名
- **handle**: 产品 handle（Shopify URL 路径）

### Google Trends 数据
- **trends_score**: 0-100 分热度评分（基于 AI 评分转换）
- **trend_direction**: 趋势方向（上升↗ / 平稳→ / 下降↘）
- **yoy_growth**: 同比增长百分比（正数为增长，负数为下降）
- **seasonality**: 季节性判断（是/否）

### FB 广告数据
- **active_ads_count**: 当前活跃广告数量
- **ad_duration**: 广告持续天数（估算）
- **unique_domains**: 投放独立站数量
- **creative_type**: 素材类型（图片/视频）

### AI 智能判断
- **ai_score**: 0-100 分综合评分
- **follow_decision**: 跟品建议（🟢立即跟 / 🟡观察）
- **risk_level**: 风险等级（低/中/高）
- **profit_margin**: 利润空间预估
- **market_saturation**: 市场饱和度（低/中/高）
- **supply_chain**: 供应链难度（易/中/难）

### 增强字段
- **product_type**: 产品类目
- **vendor**: 品牌/供应商
- **tier**: 评级层级
- **ad_grade**: 广告评级
- **first_seen**: 首次发现时间
- **price_range_low**: 竞品价格带下限
- **price_range_high**: 竞品价格带上限

## 使用流程

### 标准工作流程

1. **访问页面**: 打开 `http://127.0.0.1:8001/v3`

2. **加载数据**: 
   - 点击"快速加载"按钮（从本地 snapshot 读取）
   - 或点击"刷新扫描"按钮（实时访问 Shopify API）

3. **切换到可跟品产品**:
   - 点击"🎯 可跟品产品"标签
   - 系统自动加载可跟品产品列表

4. **调整筛选条件**:
   - 修改"最低评分"输入框（例如改为 50）
   - 点击"刷新"按钮重新加载

5. **查看产品详情**:
   - 点击产品名称链接，在新标签页打开产品页面
   - 查看 AI 评分、Trends 数据、广告数据等

6. **生成 AI 调研**:
   - 点击产品行的"AI 调研"按钮
   - 等待提示词生成（约 1-2 秒）
   - 按钮显示"✓ 已复制"表示成功
   - 粘贴到 Claude 对话框进行深度分析

7. **分析决策**:
   - 根据 AI 返回的 8 维度分析结果
   - 参考跟品决策建议（立即跟/观察/不跟）
   - 查看风险提示和策略建议
   - 做出最终跟品决策

## 技术实现

### 前端
- **框架**: 原生 JavaScript + HTML + CSS
- **样式**: 深色主题，响应式设计
- **交互**: 标签页切换、异步数据加载、剪贴板 API

### 后端
- **框架**: FastAPI (Python)
- **数据源**: 
  - Shopify snapshot 文件（本地快照）
  - V3 缓存文件（扫描结果）
  - FB Ad Library 数据（8000 端口服务）

### 数据流
1. `today-fast` API 从 snapshot 文件读取今日新品
2. `_v3_score_all_products` 对产品进行评分
3. `followable-products` API 筛选高分产品并增强数据
4. `generate-research-prompt` API 生成调研提示词

## 注意事项

1. **Google Trends 数据**: 当前为模拟数据（基于 AI 评分转换），实际部署时应接入真实的 Google Trends API

2. **评分系统**: AI 评分基于价格信号、广告数据、品类等多维度计算，评分范围 0-100

3. **数据时效性**: 
   - "快速加载"使用本地 snapshot 数据（秒级响应）
   - "刷新扫描"实时访问 Shopify API（10-30 秒）

4. **浏览器兼容性**: 需要支持 Clipboard API 的现代浏览器（Chrome 63+, Firefox 53+, Safari 13.1+）

5. **数据依赖**: 
   - 需要先运行"快速加载"或"刷新扫描"生成数据
   - 需要 8000 端口的 FB Ad Library 服务运行中

## 故障排查

### 问题：点击"刷新"后显示"暂无数据"
**解决方案**: 
1. 先点击页面顶部的"快速加载"按钮
2. 或点击"刷新扫描"按钮生成数据
3. 确认 snapshot 文件存在于 `output/shopify_monitor/` 目录

### 问题：AI 调研按钮点击后显示"失败"
**解决方案**:
1. 检查产品链接是否正确
2. 确认服务器正常运行（8001 端口）
3. 查看浏览器控制台错误信息

### 问题：评分都很低，没有高分产品
**解决方案**:
1. 降低"最低评分"阈值（例如改为 40）
2. 点击"刷新"重新加载
3. 评分系统基于多维度计算，低分可能表示当前市场机会较少
